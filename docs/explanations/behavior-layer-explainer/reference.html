G1 trigger rules

When does editing a test require an INTENT-NNN record? Keyed off the git status of accepted behaviors' locators, using the behavior's state at HEAD:

Locator changeBehavior accepted at HEAD?Record required?
Modifiedyesyes — "regardless of what" the edit is; no cosmetic exemption
Deletedyesyes — removing a live guarantee is an intent change
Added (brand-new behavior)yesno — the normal accept flow, not a change to an existing guarantee
Pure rename (no content change)yesno — moving a file isn't changing the guarantee; a rename with edits counts as modified
any changeno (proposed/confirmed/quarantined/deprecated)no — only accepted guarantees are governed
Escape valves & the honest blind spot

Quarantining or deprecating the behavior in the same change as its test edit exits the gate (not accepted at HEAD ⇒ no record needed). And G1 is path-based: edits to a shared test helper that isn't the locator aren't caught — an accepted limitation; a future refinement could widen detection to the test's import closure. With no baseline marker yet (fresh repo), the gate skips rather than false-blocks.

Glossary

Behavior
an intended, observable fact about the system: a stable record (BEH-NNN) with a lifecycle, linked via an adapter to the test that verifies it. Not tied to any one test format.
Adapter / locator
how a behavior points at its verification (Gherkin scenario, native test, or manual) and where that test lives.
Lifecycle
proposed → confirmed → accepted, with quarantined/deprecated off to the side. Only accepted, non-quarantined behavior is authoritative.
Coverage fingerprint
the set of files a single test executes — the provenance-bearing TEST → CODE edge, never asserted as precise.
Provenance / trust order
the trust order is explicit (hand-declared — reserved as a future lever, not emitted today) > observed > static; every shipped edge carries observed or static, plus a confidence and a freshness commit. Higher trust wins on merge.
Coverage-unknown
an edge the graph knows it should resolve but couldn't — surfaced with a reason, never dropped. Distinguishes "no deps" from "couldn't resolve."
Direction A / B
A = code change → affected behaviors (regression). B = a behavior → the code that implements it (planning).
Generated projection
a derived, read-only view of a single source of truth (allowed). Distinct from a hand-maintained duplicate (forbidden).
Declared-intent record
a durable, machine-checkable artifact (INTENT-NNN: behaviors touched, rationale, approver) — the only legitimate reason to change an accepted behavior's test.
Principle / ADR
principles.md = the project constitution (highest authority); an ADR (ADR-NNN) = a cross-cutting architecture decision. Authority: principle > ADR > spec.
Resolve-to-proceed
a model finding must be resolved (fix / refute / amend / reconcile) before wrap-up completes — "ignore and push" is not a resolution.
Behavioral blast-radius
the set of intended behaviors affected by a change.
