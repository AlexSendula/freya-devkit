Security finding statuses

The prose report uses eight statuses; the machine-readable findings.json collapses them to three (open / resolved / intentional):

Prose statusfindings.json
CONFIRMED · MITIGATED · NEEDS REVIEW · PERSISTENT · REGRESSEDopen
RESOLVEDresolved
INTENTIONAL DESIGNintentional
FALSE POSITIVEexcluded (not in report)

Consumers treat any finding whose status is not open as not outstanding. intentional is backed by a spec_ref (a prose claim) or the stronger behavior_ref (an accepted, test-backed guarantee).

Glossary

Blast radius
A file plus everything that transitively depends on it — computed by code-graph. What a change actually reaches.
Internal edge
An import that resolved to a real project file (not external: or unresolved:). The signal for "real feature wiring."
Behavior (BEH-NNN)
A first-class, intent-bearing record of an intended behavior, linked to a test. Numbered stably, never renumbered.
Lifecycle state
proposed → confirmed → accepted (+ quarantined/deprecated). Trust is the state; only accepted gates.
Adapter
How a behavior links to a test — a Gherkin scaffold (writes a skeleton .feature) or a native link to an existing test.
Locator
The address of a behavior's test, e.g. path#scenario-slug or path::node.
Entry
The route/handler file an integration behavior drives; its code-graph closure is the static coverage.
Fingerprint
The TEST → CODE record of which files a behavior's test exercised, with a coverage value and confidence.
Certainty
A 0–100 confidence score on an inferred spec. A heuristic, not the trust signal for executable behavior.
INTENT-NNN
A declared-intent record authorizing a change to an accepted behavior's test (governance G1).
ADR
Architecture Decision Record — a cross-cutting decision in decisions/. Outranks a spec, below a principle.
Principle
A project-wide rule in principles.md — the constitution, highest authority.
Resolution log
An append-only JSONL recording how a resolve-to-proceed finding was settled, so it isn't re-litigated.
Two-commit pattern
Code changes in one commit, generated artifacts in a second — clean history, stable references.
Validate-on-hit
wrap-up surfacing the proposed/confirmed behaviors a change touched (and uncovered code) — advisory, skippable.
Never-synced guard
wrap-up's refusal to let incremental update silently full-generate on a project that was never synced.
